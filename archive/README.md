# Archive

**Completed projects and historical records.**

Use this folder to store projects that are finished, closed, or no longer active. Archiving keeps your active workspace clean while preserving history.

## When to Archive

Move a project to archive when:
- Deliverable is complete and accepted
- Project is cancelled or paused indefinitely
- It has been in "done" state for >30 days

## Format

Preserve the original structure:
```
archive/
  YYYY/
    project-name/
      README.md        (final status, completion date)
      notes.md
      assets/
    another-project/
```

## Archival Checklist

Before archiving:
- [ ] Final status documented
- [ ] Key learnings captured in project README
- [ ] Files organized and named clearly
- [ ] Links/dependencies noted

## Retrieval

Periodically review archive:
- Extract patterns and lessons learned
- Reuse successful templates or processes
- Celebrate completed work!

Archived projects may be reopened if needed.
