# Internationalization (i18n) Implementation

**Date:** 2026-03-01  
**Status:** ✅ Complete  
**Bugs Fixed:** 1.1, 1.2

## Overview

This document describes the lightweight internationalization (i18n) system implemented for the Wizdi platform to support Hebrew and English languages.

### Key Features

- **No external dependencies** — Uses pure JSON and React hooks
- **Environment-based language selection** — Set via `REACT_APP_LANGUAGE` at build time
- **Fallback mechanism** — Missing translations gracefully return key name or default value
- **Type-safe** — Written in TypeScript
- **Accessible** — Supports ARIA labels and descriptions in both languages

---

## Architecture

### Translation Files

Translation files are stored as simple JSON objects in `/src/locales/`:

```
src/locales/
├── he.json          # Hebrew translations
├── en.json          # English translations
└── useTranslation.ts # Custom React hook
```

Each file contains a nested structure:

```json
{
  "exam": {
    "upload": {
      "title": "...",
      "description": "...",
      "error": {
        "selectFile": "..."
      }
    }
  }
}
```

### Custom Hook: `useTranslation`

The `useTranslation` hook provides a simple `t()` function for accessing translations:

```typescript
import { useTranslation } from '../../locales/useTranslation';

export default function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <h1>{t('exam.upload.title')}</h1>
  );
}
```

**Function Signature:**

```typescript
function t(key: string, defaultValue?: string): string
```

- `key` — Dot-separated path to translation (e.g., `'exam.upload.title'`)
- `defaultValue` — Optional fallback value if translation not found
- **Returns** — Translated string, default value, or key name as fallback

### Language Selection

Language is determined at **build time** by the `REACT_APP_LANGUAGE` environment variable:

```bash
# Build for Hebrew (default)
npm run build
# or explicitly
REACT_APP_LANGUAGE=he npm run build

# Build for English
REACT_APP_LANGUAGE=en npm run build
```

The `useTranslation.ts` file reads this at compile time:

```typescript
const LANGUAGE = (process.env.REACT_APP_LANGUAGE || 'he') as 'he' | 'en';
```

## Usage Examples

### Basic Translation

```typescript
import { useTranslation } from '../../locales/useTranslation';

function ReferenceExamUpload() {
  const { t } = useTranslation();
  
  return <h2>{t('exam.upload.title')}</h2>;
  // Renders: "העלאת מבחן ייחוס" (Hebrew) or "Reference Exam Upload" (English)
}
```

### With Default Value

```typescript
const message = t('custom.key', 'Fallback Text');
// If 'custom.key' doesn't exist, returns 'Fallback Text'
```

### ARIA Labels for Accessibility

```typescript
<input
  aria-label={t('exam.accessibility.selectFile')}
  aria-describedby={t('exam.accessibility.fileHelp')}
/>
```

### Conditional Subject Labels

```typescript
const SUBJECTS = [
  { value: 'math', label: t('exam.subject.math') },
  { value: 'hebrew', label: t('exam.subject.hebrew') },
  { value: 'english', label: t('exam.subject.english') },
];
```

## Fallback Behavior

If a translation key is missing, the system handles it gracefully:

### 1. Key Exists → Return Translation
```typescript
t('exam.upload.title')  // Returns translated string
```

### 2. Key Missing, Default Provided → Return Default
```typescript
t('missing.key', 'Default Text')  // Returns 'Default Text'
```

### 3. Key Missing, No Default → Return Key Name + Console Warning
```typescript
t('missing.key')  // Returns 'missing.key'
// Console: [i18n] Missing translation: missing.key
```

This makes it easy to identify missing translations during development.

### Future Expansion

To add a new language (e.g., Arabic):

1. Create `src/locales/ar.json` with all keys
2. Update `useTranslation.ts`:
   ```typescript
   import arTranslations from './ar.json';
   
   const translations: Record<'he' | 'en' | 'ar', Record<string, any>> = {
     he: heTranslations,
     en: enTranslations,
     ar: arTranslations,
   };
   
   const LANGUAGE = (process.env.REACT_APP_LANGUAGE || 'he') as 'he' | 'en' | 'ar';
   ```
3. Build: `REACT_APP_LANGUAGE=ar npm run build`

## Bugs Fixed

### Bug 1.1: Hardcoded Hebrew Labels
**Issue:** All UI labels were hardcoded in Hebrew, making the app impossible to use in English.  
**Solution:** Moved all strings to `src/locales/he.json` and `src/locales/en.json`.

**Files Updated:**
- `src/components/referenceExams/ReferenceExamUpload.tsx`

### Bug 1.2: Error Messages Without Translation
**Issue:** Error messages were hardcoded, preventing error communication in other languages.  
**Solution:** All error messages now use the translation system with keys under `exam.upload.error.*`.

**Example:**
```typescript
// Before
setError('יש לבחור קובץ');

// After
setError(t('exam.upload.error.selectFile'));
```

## Maintenance

### Adding New Translations

1. **Identify the missing string** in the component
2. **Create a logical key** (e.g., `exam.upload.newString`)
3. **Add to both `he.json` and `en.json`**
4. **Replace hardcoded string** with `t('exam.upload.newString')`
5. **Test both builds** to verify

### Keeping Files in Sync

Both translation files must have **identical keys**. Use this check:

```bash
# Compare JSON keys
jq 'keys' src/locales/he.json > he-keys.json
jq 'keys' src/locales/en.json > en-keys.json
diff he-keys.json en-keys.json
```

### Missing Translation Review

Check browser console during development for `[i18n]` warnings:

```
[i18n] Missing translation: exam.upload.unknownKey
```

This helps catch missing translations before production.

## Performance

- **No runtime cost** — Language is determined at build time, not runtime
- **No external API calls** — All translations are bundled with the app
- **Small bundle size** — JSON files are ~2KB each
- **Tree-shakeable** — Unused translation keys are not included in production builds

## Accessibility

All ARIA labels and descriptions are translated:

```typescript
<input
  aria-label={t('exam.accessibility.selectFile')}         // "בחר קובץ PDF של מבחן"
  aria-describedby={t('exam.accessibility.fileHelp')}     // "קובץ PDF בגודל עד 50MB"
/>
```

Screen readers will announce translated labels based on the browser's language setting (assuming the correct build is served).
