# i18n System Design — Hebrew + English

**Date:** 2026-03-01  
**Approach:** Simple JSON + Custom Hook  
**Scope:** ReferenceExamUpload.tsx (Bugs 1.1 & 1.2)  
**Future:** Can extend to entire app later

---

## Overview

Implement lightweight i18n using JSON translation files and a custom React hook. No new dependencies. Fixed at build time (rebuild for different language).

### Custom Hook: useTranslation()
```typescript
// src/locales/useTranslation.ts

const LANGUAGE = process.env.REACT_APP_LANGUAGE || 'he';
const translations = { he: heJSON, en: enJSON };

export function useTranslation() {
  const t = (key: string, defaultValue?: string): string => {
    const keys = key.split('.');
    let value: any = translations[LANGUAGE];
    
    for (const k of keys) {
      value = value?.[k];
    }
    
    return value || defaultValue || key;
  };
  
  return { t };
}

// Usage in component:
const { t } = useTranslation();
<h2>{t('exam.upload.title')}</h2>
<label>{t('exam.subject.label')}</label>
```

### Build-Time Language Selection
```bash
# Build for Hebrew (default)
npm run build

# Build for English
REACT_APP_LANGUAGE=en npm run build
```

## Data Flow

### Before (Hardcoded)
```
Component
  ↓
HTML: { label: 'מקצוע' }
```

### After (i18n)
```
Component
  ↓
useTranslation() hook
  ↓
Looks up 'exam.subject.label' in JSON
  ↓
Returns: 'מקצוע' (if Hebrew) or 'Subject' (if English)
  ↓
HTML renders correct language
```

## Error Handling

If translation key is missing:
- **Development:** Logs warning to console
- **Production:** Displays key as text (e.g., "exam.upload.title")
- **Fallback:** Accepts optional default value

```typescript
t('exam.upload.title', 'Default Title')
// Returns translation or "Default Title" if not found
```
